# Регрессия после редизайна выбора услуг (ServiceListPicker)

Полная повторная приёмка после замены dropdown на карточки + поиск + чипы.  
**Зоны продукта:** `/quick-booking`, блок записи на `/reception`, модалка новой записи в CRM (`BookingModal`).

Связанный чеклист (сценарии A–D, нижняя панель расписания): [QUICK_BOOKING_QA.md](./QUICK_BOOKING_QA.md).

Отмечайте чекбоксы при прогоне; отклонения — внизу по шаблону.

---

## 1. Окружение и уровень «release candidate»

- [ ] Сборка как на проде (или стейдж с тем же билдом).
- [ ] Реальные данные: `service_listings`, графики, записи (не мок).
- [ ] Языки: **ru** и **et** (строки `servicePicker.*`, `quickBook.*`).
- [ ] Устройства: **планшет landscape (приоритет)**, portrait, десктоп.
- [ ] Два окна/два пользователя для гонок на слот (где применимо).

---

## 2. Новый UI выбора услуг (общий компонент)

Прогнать на **каждом** экране ниже (разделы 3–5).

### 2.1 Карточки и состояние

- [ ] Все услуги из ожидаемого набора отображаются; длительность и цена (или «цена в прайсе») читаемы.
- [ ] **Выбранная** услуга: заметная подсветка, галочка, без «залипания» после смены шага.
- [ ] Длинное название: перенос/обрезка без поломки вёрстки (проверить 1–2 самых длинных наименования).

### 2.2 Поиск

- [ ] Фильтрация **по вводу без задержки** (текущая реализация — без debounce; на слабом планшете при лагах зафиксировать в баге).
- [ ] Поиск по **фрагменту названия** и по **категории** (где есть группировка).
- [ ] **Пустой результат:** сообщение `servicePicker.noMatch`, UI не ломается.

### 2.3 Чипы фильтров

- [ ] **Все** — сброс ограничений чипа (поиск может оставаться).
- [ ] **Недавние** — только если в `localStorage` уже есть история; порядок соответствует недавним выборам.
- [ ] **Частые** — сортировка по счётчику выборов; первый визит без истории не падает.
- [ ] **До 45 мин** / **От 75 мин** — множества услуг ожидаемы.
- [ ] **Дороже** — неактивен/бесполезен только когда **нет цен** в данных; с ценами отрабатывает осмысленно.

### 2.4 Скролл и касания

- [ ] Вертикальный скролл списка плавный; нет «рваного» UI при быстром скролле.
- [ ] **Hit area** карточки достаточна; редкие промахи пальцем — зафиксировать.
- [ ] Отступы между карточками не вызывают двойных срабатываний.

### 2.5 localStorage (раздельные ключи)

- [ ] Быстрая запись, ресепшен и модалка **не перетирают** чужую статистику (разные `storageKey`).
- [ ] При очистке сайта — пикер ведёт себя предсказуемо (нет ошибок в консоли).

---

## 3. Quick Booking (`/quick-booking`)

- [ ] **Категория** → **Услуги:** список только из выбранной категории; группировка по категории отключена (одна категория).
- [ ] Выбор услуги → переход к шагу мастера; **назад** с услуги не оставляет битого состояния.
- [ ] Нижняя панель расписания: после выбора услуги фильтр мастеров и длительность слотов корректны.
- [ ] Повторить сценарии **A–D** по [QUICK_BOOKING_QA.md](./QUICK_BOOKING_QA.md) целиком.

---

## 4. Reception — блок записи (`/reception`)

- [ ] Список услуг — карточки, не `<select>`.
- [ ] **Группы по категориям** отображаются; заголовки читаемы.
- [ ] При выборе мастера: **★** на услугах мастера; подписи «отфильтровано / помечено» как раньше по смыслу.
- [ ] Смена услуги сбрасывает мастера «любой» и выбранный слот (как при смене в старом select).
- [ ] Ограничение набора услуг **панелью мастеров** сохраняется.
- [ ] Создание записи: `source` / видимость в CRM / календарь — как в регрессе до редизайна.

---

## 5. CRM — модалка записи (`BookingModal`)

- [ ] Нет мелкого «quick row» из 8 кнопок и нет dropdown услуг — только пикер.
- [ ] **Компактный** режим: высота списка ограничена, скролл внутри модалки работает.
- [ ] `lockStaff` и смена услуги: список допустимых услуг и мастеров согласованы.
- [ ] Сохранение записи, пересечения, валидация — без регрессии.

## 7. Визуальная приёмка

- [ ] Выравнивание: поиск, чипы, карточки, группы.
- [ ] Типографика: заголовок услуги крупнее мета-строки (длительность · цена).
- [ ] Тени / hover / active согласованы с тёмной темой.
- [ ] Нет налезания sticky-поиска на контент при скролле (критично на планшете).

---

## 8. Ошибки и нагрузочное поведение UI

- [ ] Нет доступных слотов — сообщения и переходы как раньше.
- [ ] Нет услуг у мастера / пустой каталог — `modal.noServices` или аналог виден.
- [ ] Быстрые повторные нажатия по карточке — одна услуга выбрана, без дребезга состояния.
- [ ] Назад из визарда / закрытие модалки — нет «зависшего» оверлея или фокуса в поиске.

---

## 9. Краевые случаи данных

- [ ] Услуги без цены в БД — отображение «цена в прайсе» / эквивалент.
- [ ] Длительность 0 или нестандартная — карточка не ломается.
- [ ] Дубликаты имён (разные id) — обе строки выбираются корректно.
- [ ] Очень большой каталог + длинный поисковый запрос — см. раздел 6.

---

## 10. Реалистичный сценарий «как в салоне»

Один прогон без шпаргалки (ресепшен или quick booking):

- [ ] Разговор с «клиентом» (роль) + поиск услуги по названию.
- [ ] Смена чипа («быстрые» / «недавние») при необходимости.
- [ ] Выбор мастера и времени, просмотр календаря/слотов.
- [ ] Создание записи с пометкой **`ТЕСТ-N — automated check, не обрабатывать`**.
- [ ] Проверка в CRM: список записей + календарь.

---

## 11. Финальный gates перед показом заказчику

- [ ] Нет битых состояний после редизайна услуг.
- [ ] Нет регрессии бронирования и синка CRM.

- [ ] Нет пересечений записей из-за UI (логика слотов не ослаблена).
