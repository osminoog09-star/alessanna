# Quick Booking — pre-release QA

Чеклист для приёмки **Быстрой записи** (`/quick-booking`) и нижней панели **«Живое расписание»**. Использовать перед демо заказчику; отмечать пройденное и фиксировать отклонения по шаблону бага внизу.

После редизайна выбора услуг дополнительно пройти полный регресс по **[REGRESSION_QA_SERVICE_PICKER.md](./REGRESSION_QA_SERVICE_PICKER.md)** (ресепшен, модалка CRM, производительность, планшет).

## Окружение

- [ ] Локально или стейдж: CRM, Supabase, реальные графики и записи (не мок).
- [ ] Планшет landscape (основной), portrait, десктоп.
- [ ] Язык интерфейса: ru / et.
- [ ] Два браузера или окна: два админа одновременно (гонки на слот).

---

## Сценарии бронирования (A–D)

### A — Конкретный мастер → ближайшее время

- [ ] Выбрать услугу → конкретный мастер → «Ближайшее время».
- [ ] Слот существует, не в прошлом, совпадает с сеткой CRM.
- [ ] Подтверждение: запись в БД, `source: reception`, `created_by_staff_id` заполнен.
- [ ] Запись видна в CRM / ресепшене / календаре (день).

### B — Конкретный мастер → день

- [ ] Выбрать день в календаре визарда; занятые интервалы недоступны.
- [ ] Нельзя выбрать пересечение с существующей записью.
- [ ] После создания — конфликт при повторной попытке того же слота.

### C — Любой мастер → ближайшее время

- [ ] «Любой свободный» + ближайшее время: подставляется реально свободный слот одного из допустимых мастеров.
- [ ] В подтверждении отображается корректный мастер.

### D — Любой мастер → день

- [ ] Метки дней (свободно / плотно / выходной) согласованы с нижней панелью.
- [ ] Выбор времени не предлагает мастера без услуги.

---

## Нижняя панель «Живое расписание»

- [ ] По умолчанию режим **7 дней**; переключение День / 3 / 7 / Месяц / Квартал без лагов.
- [ ] До выбора услуги: отображаются мастера ресепшена; клик по ячейке показывает подсказку «сначала услуга».
- [ ] После выбора услуги: в матрице только мастера по услуге; длительность слота = услуга + буфер.
- [ ] Клик по свободной ячейке: подставляются дата, мастер, время; открывается шаг «Время» визарда.
- [ ] Подсветка колонки совпадает с текущим `bookYmd` визарда.
- [ ] Инсайты (сумма окон, ближайшая запись, «свободнее всех») правдоподобны на тестовых данных.
- [ ] Навигация периода и «к ближайшей дате» не ломают состояние визарда.

---

## CRM и календарь

- [ ] Новая запись в списках записей.
- [ ] Видимость в календаре: день / неделя / месяц (по возможности 3 дня и квартал, если есть в продукте).
- [ ] Цвета/мастер совпадают с назначением.
- [ ] После создания нижняя панель после обновления данных показывает занятость (F5 или dev reload в DEV).

---

## Клиентский шаг

- [ ] Поиск клиента (имя / телефон), автоподстановка.
- [ ] Новый клиент без поиска.
- [ ] Повторная запись того же клиента.

---

## Планшет и доступность

- [ ] Крупные зоны нажатия, читаемость `text-lg+`.
- [ ] Landscape: нет обрезания таблицы без горизонтального скролла там, где задуман скролл.
- [ ] Контраст легенды (зелёный / янтарный / красный / серый) и подписей.

---

## Ошибки и краевые случаи

- [ ] Нет сети / таймаут: сообщение, не зависший UI.
- [ ] Пусто: нет мастеров на панели ресепшена.
- [ ] День полностью занят; последнее окно дня; окно вплотную до перерыва.
- [ ] Мастер не работает в этот день (выходной).
- [ ] Двойной клик «Подтвердить» не создаёт дубликат (или сервер отклоняет — показ перекрытия).
- [ ] Вставка записи другим админом между выбором слота и подтверждением: overlap / сообщение.

---

## Производительность

- [ ] Переключение режимов панели и недели ≤ ~1 с на реальных данных.
- [ ] Нет заметных фризов при скролле сетки слотов визарда.
